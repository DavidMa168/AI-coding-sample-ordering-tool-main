# 內部點餐工具實作計畫

本計畫旨在根據 `提詞.md` 的需求，開發一套不依賴前端框架 (Vue/React) 以及後端代理的純前端點餐系統。系統將直接與 Google Sheets API 溝通。

## User Review Required

> [!IMPORTANT]
> 1. 由於文件中未提供**Google OAuth Client ID**與**Spreadsheet ID**，我將在 `all.js` 中保留常數變數供您填寫。
> 2. 將使用最新的 Google Identity Services (GIS) 來處理登入與取得 Access Token，以便在無後端情況下讀寫 Google Sheets。請確認您能夠在 GCP Console 中設定正確的 Authorized JavaScript origins。

## Proposed Changes

### 核心檔案

#### [NEW] index.html
- 載入 Google Identity Services 腳本
- 載入字體 (Google Fonts: Noto Sans TC) 及圖示
- 建立登入畫面 (Google 登入按鈕)
- 建立主畫面架構：
  - 管理員區塊 (設定今日餐廳、清除訂單)
  - 點餐區塊 (餐廳清單與餐點卡片)
  - 訂單確認區塊 (今日訂單列表與一鍵複製按鈕)
- 設計明亮風格的餐廳背景結構

#### [NEW] all.css
- 定義 CSS 變數（明亮主色調、圓角、陰影）
- 加入背景圖 (使用 Unsplash 餐廳相關高畫質圖片) 並加上毛玻璃 (backdrop-filter) 效果
- RWD 手機版適應設計
- 漂亮的卡片 UI 設計、微動畫與載入狀態的樣式

#### [NEW] all.js
- **環境設定：** 定義 Client ID、Spreadsheet ID 和 API Scopes。
- **身分驗證模組：** 使用 `google.accounts.oauth2.initTokenClient` 完成使用者登入並取得可用於 Sheet API 的 Token，同時抓取使用者 Email，接著查詢 `Users` 工作表比對權限（管理員 / 一般成員 / 未獲授權）。
- **Google Sheets API 模組：** 封裝 `fetch` 呼叫 Google Sheets v4 API，包含 `batchGet` 讀取資料與 `append`/`clear` 寫入和刪除資料。
- **管理員功能：** 
  - 從 `Menu` 撈出不重複餐廳清單供選擇設定。
  - 將設定資料寫入 `TodayConfig` 表格。
  - 清空 `Orders` 表格功能的實作。
- **使用者點餐功能：** 
  - 讀取 `TodayConfig` 並由 `Menu` 過濾出今日餐點。
  - 渲染點餐卡片、處理點餐送出將資料寫入 `Orders` 工作表並顯示提示。
- **訂單檢視功能：** 將 `Orders` 的現有資料撈出並提供「一鍵複製成文字」功能，並在清單明細底部自動加上動態的「金額合計」區塊。

## Verification Plan

### Manual Verification
1. 以 Live Server 開啟 `index.html`。
2. 填寫 Client ID 與 Spreadsheet ID。
3. 點擊登入授權。
4. 驗證權限是否依據試算表 `Users` 生效。
5. 管理員身份需能看到設定餐廳及清空訂單按鈕，並能正常操作寫入表格。
6. 一般使用者應能看到過濾後的點餐介面，輸入備註並點擊成功寫入訂單。
7. 檢查 CSS 是否符合明亮美觀且具備良好 RWD 體驗。
