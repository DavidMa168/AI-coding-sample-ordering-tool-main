請協助我建立一個「公司內部點餐工具」，使用純 JavaScript、HTML5 與 CSS3。
不需要使用 Vue 或 React 等框架，以保持結構簡單易懂。

## 技術需求
1. 身分驗證：使用 GCP Login 進行登入，取得使用者的 Email。
2. 資料庫連結：直接使用 Google Sheets API。不要使用 GAS 或任何第三方代理。用戶也會另外獲得本試算表的授權
3. 權限與流程邏輯：
   - 使用者登入後，需先比對 `Users` 工作表。
   - 若 Email 不在名單內，顯示「未獲授權」。
   - 若在名單內，讀取其「權限」欄位（管理員/一般成員）。

# 網頁功能要求
1. 登入介面：顯示一個明顯的 Google 登入按鈕，本專案不會使用到後端的 secret key。
2. 身分檢查：
   - 登入成功後，檢查 Email 是否在 Users 表中。
   - 在名單內者，根據其權限顯示對應功能。
3. 設定今日餐點：
   - 從 [Menu] 撈出餐廳名稱的列表
   - 僅有管理員能夠設定今天選用的餐廳（可複選）
   - 僅有管理員能夠設定今天選用的餐別
   - 餐別為下拉選單，有「午餐」、「下午茶」、「晚餐」三種選項，預設為「午餐」
   - 將餐廳設定至 [TodayConfig]
   - 點擊「儲存」按鈕後，將餐廳設定至 [TodayConfig]
4. 點餐介面：
   - 讀取 [TodayConfig] 取得今日餐廳名稱。
   - 從 [Menu] 中過濾出「屬於今日餐廳」的餐點。
   - 每個餐點卡片包含：品名、單價、備註輸入框、點餐按鈕。
5. 送出訂單：
   - 點擊後將資料（含當下時間、Email、餐別、餐廳、餐點、金額與備註）直接寫入 Orders 表。
   - 成功後顯示提示訊息。
6. 確認訂單：
	 - 列出今天的餐點列表，並且提供一鍵複製，可以貼到 LINE 或其他對話工具供人確認
7. 管理員專區：
   - 若權限為「管理員」，頁面底部會出現紅色的「清空今日點餐」按鈕。
   - 點擊可彈出確認，確認後，會刪除 Orders 表中除了標題列以外的所有資料。

# 輸出格式要求
請將代碼拆分為以下三個檔案，並加入詳細的繁體中文註解：
1. index.html：包含 HTML 結構與 Google API 腳本載入。
2. all.css：設計成簡約、現代且適合手機瀏覽的風格。
3. all.js：包含所有 API 呼叫、權限判斷、資料過濾與寫入邏輯。
